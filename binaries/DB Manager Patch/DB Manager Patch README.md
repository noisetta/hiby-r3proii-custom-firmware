# HiBy R3 Pro II — Database Manager Patch Documentation

**Binary:** `hiby_player`
**SHA256:** `71a6046f17a9a74b85af291418be2953b8d9559f35817dbb38353a9c20ab80e2`
**Architecture:** MIPS32 Little Endian (Ingenic X1600E SoC)

---

## Overview

This patch adds a **"Database Manager"** entry to the device's Settings page by patching an hidden option `(Operating Instruction)`. When tapped, it opens a sub-page with two options:

- **Save Database to SD** — copies `usrlocal_media.db` from the device to the SD card
- **Copy Database from SD** — copies `usrlocal_media.db` from the SD card to the device, deletes the SD copy, then reloads the database and refreshes the UI immediately (no reboot required)

This allows loading a database generated by the companion `update_database.py` script without running a full media scan on the device.

---

## Architecture

The `hiby_player` binary does not natively support adding custom sub-pages to Settings. The solution hijacks the existing **Font Size sub-page** (`vg_listview_font_size`): when "Database Manager" is tapped, the same Font Size view is created, but a global flag tells the patched callbacks to display the DB Manager options instead of "Default"/"Big" and to execute shell commands instead of changing the font.

Five original code locations are redirected to custom functions written in a code cave (unused region of the `.text` segment).

---

## Patched Locations in Original Code

### 1. Generator redirect — `0x4e3100`

Redirects the font_size list generator to the custom wrapper.

| Address | Original | Patched |
|---|---|---|
| `0x4e3100` | `addiu $sp, $sp, -0xAC0` | `j 0x41bec0` |
| `0x4e3104` | `sw $s4, 0xAB8($sp)` | `nop` |

### 2. SET handler redirect — `0x4e3304`

Redirects the font_size value-store callback to the custom SET handler. The original code at this location compared the current value with the selected value, then called `FUN_0042bc60(0x47, value)` to save the font size setting.

| Address | Original | Patched |
|---|---|---|
| `0x4e3304` | `beq $v0, $s0, 0x4e331c` | `j 0x41bf20` |
| `0x4e3308` | `move $a1, $s0` | `nop` |
| `0x4e330c` | `jal 0x42bc60` | `nop` |
| `0x4e3310` | `addiu $a0, $zero, 0x47` | `nop` |

### 3. Font Size click fix — `0x4e17c0`

Redirects the Font Size click handler to a trampoline that clears the flag before opening the Font Size sub-page. This ensures that if the user opens Font Size (not Database Manager), the flag is always 0 and the original behavior is preserved.

| Address | Original | Patched |
|---|---|---|
| `0x4e17c0` | `lui $a2, 0x0082` | `j 0x41bfa0` |
| `0x4e17c4` | `lui $a1, 0x0082` | `nop` |

### 4. Database Manager click — `0x4e1a0c`

Replaces the `operating_instruction` case in the Settings click dispatch.

| Address | Original | Patched |
|---|---|---|
| `0x4e1a0c` | — | `jal 0x41bfd0` |
| `0x4e1a10` | — | `nop` |
| `0x4e1a14` | — | `b 0x4e1618` (branch to function epilogue) |
| `0x4e1a18` | — | `lw $ra, 0xD4($sp)` (restore return address in delay slot) |

## Code Cave Layout (`0x41bd38` – `0x41c0b8`)

### Data Strings

| Address | Length | Content |
|---|---|---|
| `0x41bd38` | 110 B | `cp /usr/data/mnt/sd_0/usrlocal_media.db /usr/data/usrlocal_media.db && rm /usr/data/mnt/sd_0/usrlocal_media.db` |
| `0x41bda8` | 67 B | `cp /usr/data/usrlocal_media.db /usr/data/mnt/sd_0/usrlocal_media.db` |
| `0x41bdec` | 10 B | `save_to_sd` (key for `sys_set.ini` lookup) |
| `0x41bdfc` | 12 B | `copy_from_sd` (key for `sys_set.ini` lookup) |

### Global Flag

| Address | Segment | Size | Purpose |
|---|---|---|---|
| `0x906100` | BSS | 1 byte | `0` = Font Size mode, `1` = DB Manager mode |

### Generator Wrapper — `0x41bec0` (88 bytes)

Replaces `FUN_004e3100`, the function that populates the font_size sub-page list. Reads the flag at `0x906100`:

- **flag = 0** — loads the original string keys `"small"` (`0x82254c`) and `"big"` (`0x82257c`) and jumps to the original generator body at `0x4e3130`
- **flag ≠ 0** — loads `"save_to_sd"` (`0x41bdec`) and `"copy_from_sd"` (`0x41bdfc`) and jumps to `0x4e3130`

The generator body reads the label text from `sys_set.ini` using these keys.

### SET Handler — `0x41bf20` (~104 bytes, two paths)

Intercepts the font_size value-store callback at `0x4e3304`. Reads the flag at `0x906100`:

**Font Size path (`0x41bf20`, flag = 0):**
- Replicates original behavior: compares current value (`$v0` from `FUN_0042a500(0x47)`) with selected index (`$s0`)
- If different: calls `FUN_0042bc60(0x47, $s0)` to save the font setting, then jumps to `0x4e3314` (calls `FUN_0044aee0` to refresh the UI)
- If same: jumps to `0x4e331c` (skip, return)

**DB Manager path (`0x41bf50`, flag ≠ 0):**
- Clears flag to `0`
- Allocates a small stack frame (saves `$ra`)
- If `$s0 = 0` (Save): calls `system("cp /usr/data/usrlocal_media.db /usr/data/mnt/sd_0/usrlocal_media.db")`
- If `$s0 ≠ 0` (Copy): calls `system("cp /usr/data/mnt/sd_0/usrlocal_media.db /usr/data/usrlocal_media.db && rm /usr/data/mnt/sd_0/usrlocal_media.db")`
- Jumps to the Reload Routine at `0x41c070`

The `system()` libc function is at `0x8e7090`.

### Font Fix Trampoline — `0x41bfa0` (~40 bytes)

Ensures Font Size works normally even if the flag was left dirty:

1. Writes `0` to the flag at `0x906100`
2. Loads the original parameters for `FUN_004aba00` (view name `"vg_listview_font_size"` at `0x829448`, data key `L"font_size"` at `0x8296b4`)
3. Calls `FUN_004aba00($s2, view_name, data_key)` to create the sub-page
4. Jumps to `0x4e17d8` to complete the original break (epilogue restoration)

### Click Handler — `0x41bfd0` (~152 bytes)

Called when the user taps "Database Manager" in Settings:

1. Sets the flag at `0x906100` to `1`
2. Calls `FUN_004aba00($s2, "vg_listview_font_size", L"font_size", 0)` to create the sub-page view — the generator wrapper will see flag=1 and populate it with DB Manager options
3. If the view was created successfully (`$v0 ≠ 0`):
   - Reads the `"operating_instruction"` label from `sys_set.ini` via `FUN_00427820` for the title bar
   - Calls `FUN_004dbf20` to set the title in the navigation bar
   - Stores the back handler at `view + 0x3b90`
4. Returns

### Reload Routine — `0x41c070` (72 bytes)

Called after both Save and Copy operations. Closes the current database connection, reopens it from the (potentially updated) file on disk, and sends UI refresh events:

1. **`FUN_00718be0()`** — `lg_music_db_done`: closes the current SQLite connection, finalizes all prepared statements, and frees the in-memory database state
2. **`FUN_00718500("MEMORY", 0x1000, "/data/usrlocal_media.db")`** — `lg_music_db_init`: reopens the database from the file on disk, reinitializes all prepared statements and in-memory caches
3. **`FUN_00459840("1#1@[01]")`** — sends the "media scan complete" event to refresh the music library UI
4. **`FUN_00459840("2#0@[06]")`** — sends a secondary UI refresh event (also sent by the firmware after a native scan)
5. Restores `$ra` and `$sp`, then returns to the SET handler's caller via `j 0x4e331c`

This is the same close-and-reopen sequence used by the firmware's own "Update Database" feature (seen at `0x4e2554` → `0x4e2580`), ensuring the reload is safe and complete.

## Execution Flows

### Opening Database Manager

```
User taps "Database Manager" in Settings
  → dispatch hits case operating_instruction (0x4e1a0c)
    → jal Click Handler (0x41bfd0)
      ├─ flag = 1
      ├─ FUN_004aba00("vg_listview_font_size", L"font_size")
      │   └─ Generator Wrapper (0x41bec0) sees flag=1
      │       └─ loads "save_to_sd" / "copy_from_sd" keys
      │           └─ reads labels from sys_set.ini
      ├─ sets title bar to "Database Manager"
      └─ sets back handler
```

### Selecting an Option

```
User taps "Save Database to SD" (item 0)
  → FUN_004e32c0 (font_size SET callback)
    → reads selected index ($s0 = 0)
    → reads current font value via FUN_0042a500(0x47)
    → j SET Handler (0x41bf20)
      → flag = 1 → DB Manager path (0x41bf50)
        ├─ clears flag to 0
        ├─ system("cp ... to SD")
        └─ j Reload Routine (0x41c070)
             ├─ lg_music_db_done()
             ├─ lg_music_db_init("MEMORY", 0x1000, "/data/usrlocal_media.db")
             ├─ event "1#1@[01]" (media refresh)
             ├─ event "2#0@[06]" (UI refresh)
             └─ return

User taps "Copy Database from SD" (item 1)
  → same path, $s0 = 1
    → system("cp ... from SD && rm ...")
    → Reload Routine (database reloaded, UI refreshed immediately)
```

### Opening Font Size (Unaffected)

```
User taps "Font Size" in Settings
  → case font_size (0x4e17c0)
    → j Font Fix Trampoline (0x41bfa0)
      ├─ flag = 0 (safety reset)
      ├─ FUN_004aba00("vg_listview_font_size", L"font_size")
      │   └─ Generator Wrapper sees flag=0
      │       └─ loads original "small" / "big" keys
      └─ j 0x4e17d8 (original epilogue)
```

## Required Configuration Files

### `set_functions.json`

Add the following entry:

```json
"operating_instruction": 1
```

This registers "Database Manager" as a clickable item in the Settings page.

### `sys_set.ini` (UTF-16LE with BOM)

Add these entries in the active language section:

```
<operating_instruction>Database Manager</operating_instruction>
<save_to_sd>Save Database to SD</save_to_sd>
<copy_from_sd>Copy Database from SD</copy_from_sd>
```

The `save_to_sd` and `copy_from_sd` keys are read by the generator wrapper to populate the sub-page list items.
